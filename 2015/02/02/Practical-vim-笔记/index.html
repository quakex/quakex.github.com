《Practical Vim》 笔记

发表于 2015-02-02   |   分类于 note   |  

《Practical Vim》 是本非常好的Vim书。好好阅读，掌握编辑器之神。

第一章 vim 解决问题的方式

tip 1 Meet the Dot Command

  • 灵活使用Dot Command。
    . 能让我们重复上次的修改,它是 vim 中最强大的命令.

The dot command lets us repeat the last change. 注意，这里是 change。当你需要执行这次改变时，用.。

这里的change 可以是字符、整行、甚至是整个文件。

G 缩进到末尾,> 是缩进，G是到文件末尾。
j. 跳到下一行，再缩进
j.
很有用的命令，下一行继续刚做出的改变。如果要对编辑行做重复的动作，用这个就对了。要常用，多用，时时用。

The Dot Command Is a Micro Macro.

change 时，要想到.

tip 2 Don’t Repeat Yourself 不要重复

在每行末尾加个;.肿么办？A ；A；A；，弱爆了，第一行用A；，后面的就用 j.

上一条的原则：能用.时，尽量用.。

A;<Esc> j. 注意这里的 esc ，完结一个操作。

; 重复上一次查找。
. 重复上一次修改。
;. 两个组合在一起那是相当拉轰。

一键移动，一键操作。 vim的精髓所在！

两个不如一个

C c$
s cl
S ^C
I ^i
A $a
o A
O ko

Tip 3 Take One Strp Back, Then Three Forward 以退为进

任务：给每个+前后加空格。
f+ 找到+号。
s空格+空格<Esc> 以退为进表现在这里，这里是退，似乎多敲了键，但为了后面的方便。实为进。
; 重复上次查找
. 重复上次修改
;. 重复上次重复

Tip 4 执行、重复、回退

有些命令以其它方式重复：
@: 可以重复任意Ex命令。
& 重复上次的:substitute 命令。

###可重复及回退的操作

{edit}            .       u
f{char}/t{char}   ;       ,
F/T               ;       ,
/pattern<CR>      n       N
?pattern<CR>      n       N
:s/target/replace &       u
qx{changes}q      @x      u

Tip 5 Find and Replace 查找和替换

全局替换 : 

:%s/查找内容/替换内容/g

按需替换： 

/查找内容  
'*'   查找当前光标下的单词。  
cw 替换内容<ESC>  
n  
.

Tip 6 结识 . 范式

理想模式：一键移动，一键执行
尽可能的用.代替修改。

第2章 Normal Mode 普通模式

普通模式命令的强大，很大程度上源于它可以把操作符与动作命令结合在一起。

Tip 7 停顿时请移开画笔

不修改时，请保持在Normal Mode.
Normal Mode 是使用vim时的常态。需要修改时才进入插入模式。

Tip 8 把撤销单元切成块

控制撤销的粒度

i{insert some text}<Esc> 是一次修改。

** 要控制好对Esc的使用，就可使撤销命令作用于单词、句子或段落。

让每个『可撤销块』对应一次思考过程。

每次停顿都是一个自然的中断点，提示我该退出插入模式，进入普通模式了。

<CR> 用 <Esc>o 取代，代表的就是这种思想。

Tip 9 构造可重复的修改

任务：反向删除一个单词
db -> x
b -> dw
daw

哪种方案最好？毫无疑问 daw，因为可以发挥.的最大威力。

aw : 文本对象

多使文本对象。

Tip 10 用次数做简单的算术运算

<C-a>   +
<C-x>   -

Tip 11 能够重复，就别用次数

d2w    2dw    dw.

三者用哪一个？dw. why？因为一是好撤销，二是撤销好控制。

执行，重复，回退 vim 的精髓。

tip 12 Combine and Conquer 双剑合壁，天下无敌

Vim 的强大很大程度上源于操作符与动作命令相结合。

操作符 + 动作命令 = 操作 

d{motion} = dl  daw  dap  

g~  gu  gU  gUaw

Vim 语法规则：当一个操作符命令被连续调用两次时，它会作用于当前行。比如dd,>> gUU

常用命令：

c   修改  
d   删除  
y   复制到寄存器  
g~  反转大小写  
>   增加缩进  
<   减少缩进  
=   自动缩进  
!   使用外部程序过滤所跨越的行。 嘛意思？

!{motion}{filter} Filter {motion} text lines through the external program {filter}.

  • 还可以自定义动作命令。
    比如textobj-entire 插件 ie ae 代表整个文件。

第3章 Insert Mode 插入模式

Tip 13 在插入模式中可即时更正错误

使用Backspace之外的键。 

Keystrokes Effect
<C-h>   Delete back one character  同BackSpace
<C-w>   Delete back one word  删除一个单词
<C-u>   Delete back to start of line 删至行首

Tip 14 返回普通模式

<Esc>       最好用软件讲caps lock 和esc 互换，mac 下可以使用Seil，免费的
<C-[>       切换到普通模式
<C-o>       切换到插入-普通模式，当我们处于插入模式时，想运行一个普通模式命令，然后马上回到原来的位置继续输入，就可以使用这种模式。  

<C-o>zz     把当前行处于屏幕中间并继续编辑，如果先退出插入模式进入普通模式再输入命令，再进入插入模式就太麻烦了，用这条命令就很实用了。

tip 15 Paste form a Register Without Leaving Insert Mode 不离开插入模式，粘贴寄存器中的文本

yt, 很好的命令，复制本行字符直到,前一个字符。要灵活使用t。

<C-r>{register} 在插入模式下的粘贴，就是<C-v>，只是灵活性上更好。

In Insert mode, we can press 0 to paste the text that we just yanked at the current cursor position.

tip 16 随时随地做运算

<C-r>=6*35<CR> 6 chairs, each costing $35, totals $210

表达式寄存器可以用来执行一段vim脚本，并返回其结果。
可以当做计算器，蛋疼啊。

用=符号指明使用表达式寄存器。在插入模式中，输入<C-r>=就可以访问这一寄存器了。

tip 17 Insert Unusual Characters by Character Code 使用字符编码插入非常用字符

<C-v>{code}
<C-v>065 打出的是 A。

<C-v>u00bf  ¿
如果你想知道文档中任意字符的编码，只需吧光标移动她上面并按`ga`命令即可，屏幕下方会显示。

tip 18 用二合字母插入非常用字符

<C-k>>{char1}{char2}
不常用，算了。

tip 19 Overwrite Existing Text with Replace Mode 用替换模式替换已有文本

R 命令可以由普通模式进入替换模式。然后可以一直替换。
和 r 略有不同。
虚拟替换模式：gR触发，用得不多。

第4章 Visual Mode 可视模式

Vim 有3种不同的可视模式，分别用于操作字符、行和块。

tip 20 深入理解可视模式

viw 选择单个词。用c命令修改所选的内容。用c而不是Backspace。

tip 21 Define a Visual Selection 选择高亮选区

v   character-wise  激活面向字符的可视模式  
V   line-wise       激活面向行的可视模式  
<C-v>   block-wise  激活面向列的可视模式  
gv      Reselect the last visual selection  重新选择上次的高亮选区   
o       Go to other end of qhighlighted text  切换选取的活动端，头尾切换

:if expand(“%”) == “”|browse confirm w|else|confirm w|endif

tip 22 Repeat Line-Wise Visual Commands 重复执行面向行的可视命令

对选中的行也可以用.
使用.命令重复一条可视模式命令时，它所操作的文本数量和上次被高亮选中的文本数量相同。

tip 23 Prefer Operators to Visual Commands Where Possible 最好使用操作符命令，而不是可视命令

可以使用vit来选择标签里的内容。it表示一种文本对象。 t是 tag 的意思。

问题：可视模式命令被重复执行时，它会影响相同数量的文本。

用qgUit，不用vitU。

如果我们想使点命令能够重复某些有用的工作，最好远离可视模式。
有时要修改的文本范围的结构很难用动作命令表达出来，可视模式可能更擅长。

prefer operator commands over their Visual mode equivalents when workong through a repetitive set of changes.

tip 24 Edit Tabular Data with Visual-Block Mode 用面向列块的可视模式编辑表格数据

<C-v>3j     选中3列
x..         删除两个空格
gv          重新选择列
r|          替换为|
yyp         复制一行
Vr-         替换为-

这里要学习的是三种 v 的选择。复习一遍：

v   character-wise 
V   line-wise
<C-v>   block-wise
gv      Reselect the last visual selection   
o       Go to other end of highlighted text

tip 25 Change Columns of Text 修改列文本

注意使用 选择列，改变一行即改变所有选择行。 

li.one   a{ background-image: url('/components/sprite.png'); }
li.two   a{ background-image: url('/components/sprite.png'); }
li.three a{ background-image: url('/components/sprite.png'); }

tip 26 Append After a Ragged Visual Block 在长短不一的高亮块后添加文本

var foo = 1;
var bar = 'a';
var foobar = foo + bar;

前面遇到过的一个需求，给每一行加一个『;』,用 Visual Block 更酷，更简单。 

<C-v>jj$
A;
<Esc>

处于插入模式期间，任何输入的内容只会出现在顶行，然而一旦返回到普通模式，这些修改就会扩散到其余被选中的行上。
