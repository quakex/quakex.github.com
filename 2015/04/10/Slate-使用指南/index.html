Slate 简介

Slate 是Mac OSX下一个键盘驱动的窗口管理软件，免费，开源，强大，唯一的缺点就是稍复杂，配置上手有点困难。和 Vim 一样，用神器要付出点小代价。但相信我，这个代价是绝对值得的。何况，还有我这样蛋疼大叔帮你开路不是么。

用 Slate 可以做哪些事

  • 移动窗口位置，改变窗口大小，布置窗口。
  • 快速切换窗口，直接快捷键 Focus 程序，绝对比 Command+tab 快得多。还有一种方法类似 easyMotion和 Vimium，程序员都懂的，这里也推荐一下Vimium插件，Chrome 必备啊。为什么 Safari 就没一个这么好用的插件呢？vimari 勉强可用，但比起Vimium还是差不少。
  • 管理多显示器。Slate 可以快速的将窗口布置到不同的显示器，这点是我最喜欢的功能。
  • 支持窗口布局。嘛意思？比如你工作时，主显示器开 Xcode，左边显示器左半部分放 Safari 浏览资料，右半部分放 iTerm；右显示器左上放iTunes，左下放 QQ 聊妹子，右半屏幕还可以用 Vim。肿么办？每次打开电脑都得拖窗口一个一个的放到固定的位置就太麻烦了，用Slate 的窗口布局，再配个快捷键，一键搞定啊，骚年！相信孤独的你一定想在心仪的妹子前表演这个酷炫的动作，从而赢得姑娘的芳心，Just do it！你将孤独一生。扯远了。
    Slate 还支持多个布局，这点是非常强的，比如你针对一个、两个或三个显示器使用三种不同的窗口布局，家里、办公、code、写作、娱乐使用不同的布局，这些布局可以随时一键切换。多么的人性化啊，人参都变得精彩了是不是？

Slate 配置文档

默认配置文件：用户目录(~/)下的.slate文件。
如果用户没有自己建立这个文件，Slate 会使用一个默认的配置文件。 这个默认的配置文件也很好用。

配置文件可以细分为以下几个方面：

  • config (全局设置)
  • alias (设置别名)
  • layout (布局)
  • default (默认布局配置)
  • bind (快捷键绑定)
  • source (配置引用)

Expressions 属性

这些属性很重要，大部分工作就是配置这些属性值。从而控制窗口的大小、位置等属性。 

screenOriginX = 屏幕原点 X 坐标
screenOriginY = 屏幕原点 Y 坐标
screenSizeX = target screen's width 屏幕宽度
screenSizeY = target screen's height 屏幕高度
windowTopLeftX = window's current top left x coordinate (should not be used in Window Hints configs) 窗口左上 X 坐标  
windowTopLeftY = window's current top left y coordinate (should not be used in Window Hints configs) 窗口左上 Y 坐标  
windowSizeX = window's width  窗口宽度  
windowSizeY = window's height  窗口高度  
newWindowSizeX = window's new width (after resize, only usable in topLeftX and topLeftY, should not be
                 used in configs)  窗口新宽度  
newWindowSizeY = window's new height (after resize, only usable in topLeftX and topLeftY, should not be
                 used in configs)  窗口新高度  
windowHintsWidth = the value of the windowHintsWidth config (only usable in windowHintsTopLeftX and
                   windowHintsTopLeftY)  窗口提示宽度，这个后面讲  
windowHintsHeight = the value of the windowHintsHeight config (only usable in windowHintsTopLeftX and
                    windowHintsTopLeftY)

操作符

+          e.g. 1+1 = 2
-          e.g. 1-1 = 0
*          e.g. 2*2 = 4
/          e.g. 4/2 = 2

还有很多，各种数学函数什么的，加减乘除就够了，一个窗口管理要 log 函数干嘛？

属性+操作符就可以描述窗口的位置了，比如 screenSizeY/2 = 屏幕宽度的一半

layout 指令

这个有点复杂，但也是 Slate 强大的地方。
语法： 

layout name 'app name':OPTIONS operations

实例： 

layout myLayout 'iTerm' push up bar-resize:screenSizeY/2 | push down bar-resize:screenSizeY/2
layout myLayout 'Google Chrome' push left bar-resize:screenSizeX/2 | push right bar-resize:screenSizeX/2

创建了一个名为myLayout的布局（layout），当你打开两个 iTerm ，触发布局后，第一个 iTerm 在屏幕的上半部分，第二个在下半部分。Chrome 同理。

Options 的说明:

Name Function
IGNORE_FAIL This will let slate move to the next operation if the current operation fails to resize/move on the current window
REPEAT This will repeat the list of operations if the number of windows is larger than the number of operations
REPEAT_LAST This will repeat the last operation in the list if the number of windows is larger than the number of operations
MAIN_FIRST This will cause the main window to always use the first operation
MAIN_LAST This will cause the main window to always use the last operation (mutally exclusive with MAIN_FIRST)
SORT_TITLE This will cause the window operations to be triggered on the windows in sorted order by the window title (can be used with MAIN_FIRST or MAIN_LAST)
TITLE_ORDER=order This will cause the operations to be triggered on the windows starting with order which is a semi-colon separated list of window titles
TITLE_ORDER_REGEX=order This will cause the operations to be triggered on the windows starting with the order which is a semi-colon separated list of window title regexes to match. Note that once a match is seen, the next regex will be used to match. This means if you have two windows that match the same regex, only the first one seen will be matched. The second will not.

看懂没有？我也是看了好久，这里的Options是什么意思？有时一个程序打开多个窗口，比如 MacVim 打开了好几个文档，这时你可以细化单独程序（例如 Macvim）的每个文档窗口的位置布局。比如屏幕左边放.cpp，右边放.h，真的是想得太周到了。只有独立开发者才会设计的这么周全，因为这都是真爱啊！

对我来说，REPEAT 和 MAIN_FIRST 就足够用了。

default 指令

语法： 

default layout-or-snapshot-name screen-configuration

当屏幕的配置发生变化时，比如一个显示器变成两个，将触发对应的 Layout。

bind 指令

可能是用得最多的指令。

语法：

bind key:modifiers operation parameter+
bind key:modal-key operation parameter+

下面讲bind指令的每个要素，特别是operation参数，注意学习。

key

键盘上的按键。什么字母就对应什么按键，小写。

modifiers 控制键

ctrl, alt, cmd, shift

Modal Key

1:f4,ctrl,alt 按下 ctrl+Alt+f4之后，激活 modal 模式，再按1就可以触发绑定了。

这种操作模式类似easyMotion，先按:后的组合键触发发大招模式，再按:前的键就可以发大招了。

Operation 操作

Screens 屏幕

指定屏幕（显示器）

  • 可以用整数值指定屏幕 ID（计数从0开始喔，你懂的），屏幕顺序从左往右，最左边的屏幕就是0，这里可以设置orderScreensLeftToRight全局变量，记得config指令不？

  • 用屏幕分辨率指定：格式 屏幕宽度x屏幕高度。

  • 相对于当前屏幕的方向：left|right|up|above|down|below。
  • next或previous。

我用的是第一种方法。

move 移动窗口

语法： 

move topLeftX;topLeftY sizeX;sizeY screen  
move 窗口左上角X坐标;左上角Y坐标 窗口宽度;窗口高度 指定屏幕

后面的命令我就不翻译了，很容易理解。太啰嗦的男人容易变弯的。

举个栗子： 

bind pad1:ctrl move 0;0 100;100 1

绑定 ctrl+numpad1，使其触发 移动当前窗口到屏幕1的左上角(0,0)处，窗口大小为（100，100）。单位都是像素。

resize

语法： 

resize x y anchor

例子： 

bind right:alt;ctrl       resize +10% +0  // 按 ctrl+alt+右方向 每次按键增加窗口宽度，大小为屏幕的10%，可以按着不松喔  
bind left:alt;ctrl        resize -10% +0  // 减小窗口宽度  
bind up:alt;ctrl          resize +0   -10%
bind down:alt;ctrl        resize +0   +10%

push

移动窗口到屏幕边缘。其实就是move指令的简化版。

语法： 

push direction style

例子： 

bind right:ctrl;cmd  push right bar-resize:screenSizeX/3 // 让窗口移动到屏幕右侧边缘，窗口宽度为屏幕的1/3；如果不指定 style 参数，默认为屏幕的1/2  
bind left:ctrl;cmd   push left  bar-resize:screenSizeX/3
bind up:ctrl;cmd     push up    bar-resize:screenSizeY/2
bind down:ctrl;cmd   push down  bar-resize:screenSizeY/2

nudge 移动窗口

# Nudge Bindings
bind right:shift;alt nudge +10% +0  // 向右移动
bind left:shift;alt  nudge -10% +0
bind up:shift;alt    nudge +0   -10%
bind down:shift;alt  nudge +0   +10%

throw 将窗口移动到指定屏幕

语法： 

throw screen style

例子： 

bind pad1:alt,ctrl throw 1 resize  // 将窗口移到屏幕1，并最大化，这里的 style 参数同 push，可以指定移动后窗口的大小

corner

语法： 

corner direction style

移动窗口到屏幕的4个角：top-left|top-right|bottom-left|bottom-right

也是move简化版。

shell 执行 shell 命令

bind 1:ctrl wait path:~/code '/opt/local/bin/mvim'

hide show toggle 隐藏、显示和切换窗口

上班看1024的利器。 

bind 1:ctrl hide 'MPlayer','Google Chrome'
bind 2:ctrl show 'MPlayer,'Google Chrome'

bind 1:ctrl toggle 'MPlayer','Google Chrome'  // 更实用

chain 星云锁链

bind 1:ctrl chain push up | push right | push down | push left

第一次按键 push up，第二次 push right，blablabla

Slate的作者实在是想得太周到了。

sequence 斗转星移

bind 1:ctrl sequence focus right > push left | push right

按ctrl+1激活当前窗口右边的窗口，同时将旧窗口 push 到屏幕左边，新窗口 push 到屏幕右边。

Jigish Patel同志，请接受我的膝盖。

layout 指定布局

bind 1:ctrl layout myLayout  // ctrl+1 激活名为 myLayout 的布局

focus 聚焦窗口

语法：focus direction|app 

bind 1:ctrl focus above  
bind 1:ctrl focus 'iTerm'

直接切换窗口焦点。用过tmux的都知道。

snapshot delete-snapshot activate-snapshot 快照功能

给当前窗口拍个快照，以便恢复。陈老师的东西，太高大上，我用不习惯。

hint 星际穿越

语法：bind 1:ctrl hint QWERTYUIOP

窗口级的easyMotion，看图：

按 ctrl+1触发hint模式，按程序图标上的字母就可以切换到相应的程序或窗口，狂拽酷炫吊炸天。

grid 网格

bind g:cmd grid padding:5 0:2,2 1:2,1
bind 1:ctrl grid padding:5 1680x1050:16,9 1050x1680:9,16

padding:5 ：grid 网格中格子线的宽度，不用修改，可忽略。

0:2,2 1:2,1: 第一个屏幕（左边屏幕）显示2x2的网格，就是把屏幕分成4份；第二个屏幕显示2x1的网格，屏幕分成了两份。

鼠标控制，类似于Moom，触发grid模式后，用鼠标控制窗口位置和大小。点哪个网格，窗口就分配到那个网格中。

知道我为什么说Slate是这些窗口管理软件的爹了吧。

relaunch

重新载入.slate文件。似乎不起作用

undo 重来

bind z:ctrl undo

执行窗口变换后，想退回去原来的状态，用undo。

我的.slate文件

# quakex 的 slate 配置文件  

# Global Configs 全局配置  
config defaultToCurrentScreen true
# show icons & letters on all windows w/ Window Hints
config windowHintsShowIcons true
config windowHintsIgnoreHiddenWindows false
config windowHintsSpread true

# Show a positioning grid for each monitor
bind g:cmd grid padding:5 0:2,2 1:2,1       #将第一个窗口分成4份，第二分成两份

# Monitor Aliases
alias mon-left 0                # 外接左边的显示器
alias mon-macbook 1             # 笔记本屏幕

# General aliases
alias hyper cmd;ctrl            # 组合键
alias sox screenOriginX 
alias soy screenOriginY
alias ssx screenSizeX
alias ssy screenSizeY

# Abstract positions  相对位置  
alias centered           ${sox}+${ssx}/8;${soy}+${ssy}/8 ${ssx}/8*6;${ssy}/8*6
alias fullscreen         ${sox};${soy} ${ssx};${ssy}
alias leftHalf           ${sox};${soy} ${ssx}/2;${ssy}
alias bottomHalf         ${sox};${soy}+${ssy}/2 ${ssx};${ssy}/2
alias topHalf            ${sox};${soy} ${ssx};${ssy}/2
alias rightHalf          ${sox}+${ssx}/2;${soy} ${ssx}/2;${ssy}
alias topLeftQuarter     ${sox};${soy} ${ssx}/2;${ssy}/2
alias topRightQuarter    ${sox}+${ssx}/2;${soy} ${ssx}/2;${ssy}/2
alias bottomLeftQuarter  ${sox};${soy}+${ssy}/2 ${ssx}/2;${ssy}/2
alias bottomRightQuarter ${sox}+${ssx}/2;${soy}+${ssy}/2 ${ssx}/2;${ssy}/2
alias topThird           ${sox};${soy} ${ssx};${ssy}/3
alias leftThird          ${sox};${soy} ${ssx}/3;${ssy}
alias bottomThird        ${sox};${soy}+${ssy}/3 ${ssx};${ssy}/3
alias rightThird         ${sox}+${ssx}/3*2;${soy} ${ssx}/3;${ssy}

# Concrete positions  具体位置  
alias 1-full        move ${sox};${soy} ${ssx};${ssy}                ${mon-macbook}
alias 1-left        move ${sox};${soy} ${ssx}/2;${ssy}              ${mon-macbook}
alias 1-right       move ${sox}+${ssx}/2;${soy} ${ssx}/2;${ssy}     ${mon-macbook}
alias 1-rightThird  move ${sox}+${ssx}/3*2;${soy} ${ssx}/3;${ssy}   ${mon-macbook}
alias 2-left        move ${sox};${soy} ${ssx}/2;${ssy}              ${mon-left}
alias 2-right       move ${sox}+${ssx}/2;${soy} ${ssx}/2;${ssy}     ${mon-left}
alias 2-righttop    move ${sox}+${ssx}/2;${soy} ${ssx}/2;${ssy}/2   ${mon-left}
alias 2-rightbottom move ${sox}+${ssx}/2;${soy}+${ssy}/2 ${ssx}/2;${ssy}/2       ${mon-left}
alias 2-rightThird  move ${sox}+${ssx}/3*2;${soy} ${ssx}/3;${ssy}   ${mon-left}

# 2 Monitor layout  两个显示器时的 layout
layout 2monitors 'iTerm':REPEAT             ${2-rightbottom} | ${1-left}
layout 2monitors 'Safari':REPEAT            ${1-full}
layout 2monitors 'Sublime Text 2':REPEAT    ${2-right}
layout 2monitors 'MacVim':REPEAT            ${2-left} | ${2-right}
layout 2monitors 'Mail':REPEAT              ${2-left}
layout 2monitors 'Finder':REPEAT            ${2-righttop}
layout 2monitors 'Xcode':REPEAT             ${1-full}
layout 2monitors 'iTunes':REPEAT            ${1-full}
layout 2monitors 'Skim':REPEAT              ${2-right}
layout 2monitors 'nvALT':REPEAT             ${2-rightThird}

# 1 Monitor layout 一个显示器时的布局  
layout 1monitor 'iTerm':REPEAT              ${1-right}
layout 1monitor 'Sublime Text 2':REPEAT     ${1-left}
layout 1monitor 'MacVim':REPEAT             ${1-left} | ${1-right}
layout 1monitor 'Safari':REPEAT             ${1-full}
layout 1monitor 'Mail':REPEAT               ${1-full}
layout 1monitor 'Finder':REPEAT             ${1-full}
layout 1monitor 'Xcode':REPEAT              ${1-full}
layout 1monitor 'iTunes':REPEAT             ${1-full}
layout 1monitor 'Skim':REPEAT               ${1-full}
layout 1monitors 'nvALT':REPEAT             ${1-rightThird}

# Default Layouts
default 2monitors resolutions:1680x1050;2560x1600
default 1monitor  resolutions:2560x1600

bind 1:${hyper} layout 1monitor
bind 2:${hyper} layout 2monitors

# Bindings 模拟 Vim 控制窗口位置  
bind g:${hyper} move ${centered}            #ctrl+cmd+g 将窗口居中
bind o:${hyper} move ${fullscreen}          #ctrl+cmd+o 全屏  
bind h:${hyper} move ${leftHalf}
bind j:${hyper} move ${bottomHalf}
bind k:${hyper} move ${topHalf}
bind l:${hyper} move ${rightHalf}
bind u:${hyper} move ${topLeftQuarter}
bind i:${hyper} move ${topRightQuarter}
bind n:${hyper} move ${bottomLeftQuarter}
bind m:${hyper} move ${bottomRightQuarter}
bind y:${hyper} move ${leftThird}
bind p:${hyper} move ${rightThird}

# Resize Bindings
bind right:alt;ctrl       resize +10% +0
bind left:alt;ctrl        resize -10% +0
bind up:alt;ctrl          resize +0   -10%
bind down:alt;ctrl        resize +0   +10%

# Nudge Bindings
bind right:shift;alt        nudge +10% +0
bind left:shift;alt         nudge -10% +0
bind up:shift;alt           nudge +0   -10%
bind down:shift;alt         nudge +0   +10%

# Throw Bindings
bind [:${hyper} throw 0 resize:${ssx};${ssy}  
bind ]:${hyper} throw 1 resize:${ssx};${ssy}  

# Window Hints
bind esc:cmd hint ASDFGQWERTZXCVBN

# relaunch .slate 
bind s:${hyper} relaunch

# Focus Bindings
bind right:${hyper}    focus right
bind left:${hyper}     focus left
bind up:${hyper}       focus up
bind down:${hyper}     focus down

# undo 
bind z:${hyper} undo

# App bindings
alias hyperApp alt;ctrl
alias editor 'MacVim'
alias browser 'Safari'
bind e:${hyperApp} focus ${browser}
bind v:${hyperApp} focus ${editor}
bind x:${hyperApp} focus 'Xcode'
bind g:${hyperApp} focus 'GitHub'
bind m:${hyperApp} focus 'Mail'
bind i:${hyperApp} focus 'iTerm'
bind f:${hyperApp} focus 'Finder'
