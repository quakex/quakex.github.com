Swift 中的 ! 和 ?

发表于 2015-03-13   |   分类于 dev   |  

看了好几遍，一直记不住，我这智商基本已经告别编程界了，伤心。还是动手写下来，免得每次想看时到处找。

？的由来

Swift 中的变量必须初始化，而且一般都得程序员自己干，编译器不会默认帮你生成一个初始值，这点和C、C++等常见语言是不同的。如果声明一个变量不手动初始化，就会报错喔。比如： 

var a:String
var b = a

但是，有些变量就是没有值啊，难不成我要写 var a = nil，这样岂不很烦，一点也不fashion。于是乎Apple的工程师就发明了Optional类型，如果没有初始化，编译器就默认将其初始化为nil。这是个特例，就有特例的做法。你需要在声明一个变量时在最后加个 ? 就表示这个变量是个Optional变量，可能为nil，也可能为非nil值。归根结底还是Optional值，确切的说是个Enum值，一下感觉好不fashion，那么swift的语言，骨子里还是奶奶级的C。 

var a : String?  // 相当于下面的语法糖
var strValue: Optional<String>

上面这个Optional的声明，意思不是”我声明了一个Optional的String值”, 而是”我声明了一个Optional类型值，它可能包含一个String值，也可能什么都不包含”，也就是说实际上我们声明的是Optional类型，而不是声明了一个String类型，这一点需要铭记在心。

Optional事实上是一个枚举类型，Optional包含None和Some两种类型，而nil就是Optional.None，非nil就是Optional.some。如果Optional变量在声明时不初始化，Swift会调用init()来初始化变量为nil，而用非nil的值初始化变量时，会通过Some(T)把该原始值包装，所以在之后使用的时候我们需要通过解包取出原始值才能使用。

a 是一个Optional值，使用Optional变量也和其它变量不同，需要解包，真麻烦，直接上不行吗？不行的，都是党员，不要那么粗鲁。需要脱衣服，想上Swift是要付出代价的！
使用Optional值的时候要在具体的操作，比如调用方法、属性、下标索引等前面加上一个?，如果是nil值，也就是Optional.None，会跳过后面的操作不执行，如果有值，就是Optional.Some，可能就会拆包(unwrap)，然后对拆包后的值执行后面的操作，来保证执行这个操作的安全性。

let hashValue = strValue?.hashValue
必须在变量名后加个问号啊。也可以不加，后面再说。

?不仅用在Optional变量上，还可以用于安全的调用protocol类型方法上。why？
因为protocol方法也可能是@optional的啊，不一定会实现，不一定的事情，就要加?，很好理解是不是？示例如下： 

@objc protocol Downloadable {
    @optional func download(toPath: String) -> Bool;
}

@objc class Content: Downloadable {
    //download method not be implemented
}

var delegate: Downloadable = Downloadable()
delegate.download?("some path") // 看这里，看这里

同样的情况在向下类型转换中也会用到——as?。

if let dataSource = object as? UITableViewDataSource {
let rowsInFirstSection  = dataSource.tableView(tableView, numberOfRowsInSection: 0)
}

到这里我们看到了?的几种使用场景:

？的使用场景

1.声明Optional值变量
2.用在对Optional值操作中，用来判断是否能响应后面的操作
3.用于安全调用protocol的optional方法
4.使用 as? 向下转型(Downcast)

都是不确定的情形。所以，?还是有道理的。

！的作用：Optional 变量的解绑 unwrap

要想对Optional 变量进行操~~~作，硬上是不行的，必须脱衣服unwrap，只有把衣服脱了才能见到我们需要的东西，那就是，真材实料、白嫩水滑的Optional.Some，而不是Optional变量本身。这里脱衣服有两种方法，一种是Optional Binding，繁复拖沓，很是不爽。又是赋值，又用if判断是否为nil。步骤如下： 

var strValue: String?

if let str = strValue {
    let hashValue = str.hashValue
}

每个Optional变量都这么用，那就太麻烦了。还有一种简洁方便的『一键脱衣法』，就是用!。 

let hashValue = strValue!.hashValue

这里的!表示“我确定这里的的strValue一定是非nil的，尽情调用吧”，一键脱衣也是有条件的，就是Optional必须为非nil，否则会出错，crash掉。

一键脱衣虽然方便，但每次都得脱也挺麻烦的，对那些我确定有货的变量，我脱一次不就够了吗？没必要每次调用时都在操作前加!啊。对于这样的变量，我们只需要在声明时脱一次就OK了。在声明变量时加!

var myLabel: UILabel!  //!相当于下面这种写法的语法糖
var myLabel: ImplicitlyUnwrappedOptional<UILabel>

这种是特殊的Optional，称为Implicitly Unwrapped Optionals, 直译就是隐式拆包的Optional，就等于说你每次对这种类型的值操作时，都会自动在操作前补上一个!进行拆包，然后在执行后面的操作，当然如果该值是nil，也一样会报错crash掉

! 的使用场景

1.强制对Optional值进行拆包(unwrap)
2.声明Implicitly Unwrapped Optionals值，一般用于类中的属性
