修复 OS X 总结

发表于 2014-03-02 | 分类于 折腾 |

我的笔记本是双系统:win7+OSX。老婆使用了两天win7后，发现不能进入Mac。变色龙正常，进入Mac系统后显示一个以前没有的进度条，然后重启。win7 MacDriver 报 “Mac volume may be damaged”，使用 Disk Genius 显示磁盘标识为07 NTFS，而不是 AF。

当时手头没有 Mac 的启动盘，血的教训，一定要准备一个启动盘！

先直接在 win7下拷贝备份的 boot，kernel，extra 等到 mac 分区，故障依旧。怀疑是分区问题。想更改 AF，被 MacDriver 禁止。

得想办法重做一个 Mac启动盘，恢复 Time Machine 。

开始制作启动盘。查询 pcbeta 远景论坛，使用 Bootdisk Utility 写入 MACPE，黑屏，不能启动。-f -x 依旧。

更改 MACPE，导入自己的 Extra，Extensions，重启出错。

显示：“Can’t find mach_kernel”，拷贝一个 kernel 还是没能解决。

安装 win 变色龙，系统不能启动，显示 Avlgo 错误。win 7也进不去。

用大白菜做了一个启动盘。

按照远景变色龙4k硬盘安装教程中的第二种方法安装 boot1h，dd。win7能启动但启动不久就直接蓝屏重启。

原因：没有仔细看变色龙安装教程，安装方法不对。安装 win 变色龙后要根据win7所在分区选择 Avlgo 文件，比如我的 win7安装在第一个硬盘的第三个分区，需要 copy 第三个文件夹中的 Avlgo 到 win7 C: 根目录替换原来的文件。这样就可以进入变色龙然后启动Win7。这里是 win 版的变色龙。

用打补丁的提取方法制作 Mac 启动盘，失败，不能启动。看 mac 分区文件，感觉不对，懒得折腾，直接下懒人版。

用老办法下载懒人版 Mac 安装盘。使用BootDisk Utility 写入。替换自己的 Extra，可以启动安装界面。

卸载 Mac Driver，这货简直就是个渣，改用HFS+。好用多了。

使用懒人版安装盘启动后运行 time Machine 恢复原来的备份，报 boot0：test boot0：error。不能启动。怀疑分区表坏。

用大白菜启动盘可以找到Mac分区进入，but 无限菊花。

-v，显示 Waiting for DSMOS 停住。

删除，替换 FakeSMC.kext，故障依旧。

time Machine 恢复失败，shit！重新安装。

重装后替换 extra。-v -f -x 可以进入，安装 Mac 版变色龙，使用最新版变色龙。少了 dd 那一步，似乎之前做过还能起作用。

使用迁移助理，效果不好，放弃，继续重装。蛋疼啊。

安装完成后，进入Mac系统声卡，网卡，触摸板，电池驱动不正常。

替换原来备份的驱动，还是不行。

怀疑变色龙可能影响驱动。（后判断应该不影响，还是驱动安装不正确）

不知道哪些驱动可以解决问题，直接替换原来的所有驱动，又进不去系统。直接停住，没有菊花，日啊！

删除 S/L/Cache/com.apple.kext.caches/Startup/ 全部文件，还是不能进入。

安装驱动，不能进入，恢复原生驱动能进入，邪门。

使用 Kext Utility 安装驱动，虽然安装较慢，而且只能一个一个的装，但是，目前这种方法有效，其它方法估计也行，但我实在是懒得折腾了。

根据文件名和 google，一个一个的试 kext 驱动。总结如下:

触摸板驱动：AppleACPIPS2Nub.kext， ApplePS2Controller.kext.

网卡，无线网卡：BCM5722D, IO80211Family, IONetworkingFamily.

声卡：AppleHDA 只有耳机有声音，继续替换 IOAudioFamily 搞定。

电池：AppleACPIPlatform.kext, AppleACPIBatteryManager, SmartBattery. 都安装了，发现第一个管用。

至此全部搞定。

总结：首先要有一个启动盘，win + mac。启动盘的制作可以用原版，但要替换内核，麻烦，还是直接下懒人版，将懒人版镜像写入安装分区或磁盘。copy extra。

安装变色龙，严格按照说明安装。

安装驱动。用kext Utility 这种最笨的方法。

后记：升级10.9.2，升级完后除了声卡不能驱动一切正常，重新安装声卡的两个 kext 后解决。我的4741g已经完美。

后后记：这篇文章是两个星期前写的，当时只是想记录一下便于以后查询。现在放在 blog 上，因为，我又决定写 blog 了，好吧，这次是真的。
黑苹果没什么技术含量，改 dsdt 也不过尔尔，关键是细心和耐心。需要不停的尝试，然后根据故障现象来判断、更改、再尝试，不停的重启修改，直至完全解决，这绝对是一件无比蛋疼的事。所以，最后我想说一句：珍爱生命，远离黑苹果！至于为什么要用黑苹果，Windows 不好吗？是的，等你用习惯了 OS X，你会觉得 Windows 确实一般。后面我会写篇文章详述，先挖个坑吧。

好了，git push，睡觉了。
