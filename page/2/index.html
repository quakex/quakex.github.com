iOS 开发的各种限制

  1. Only One Window
  2. Only One Active Application 有严格的限制，有利有弊，总体而言，利大于弊。
  3. Limited Access, sandbox 机制
  4. Limited Response Time， 假如用户按了Home键，在iOS停止你的程序前，你必须在后台迅速的保存一切。假如时间过长（5秒？），你的程序进程键被杀死，不管你是否保存完毕。这有一个API可以申请额外的时间，但建议不要用。是虾米？

  5. Limited System Resources
    Cocoa Touch 有内建的机制让你的程序侦测到『内存过低』这种情况，这时，你必须释放不需要的内存，或退出程序。
