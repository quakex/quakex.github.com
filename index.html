Sublime Text 中管理自己的TODO

发表于 2016-04-29 | 分类于 SublimeText技巧 |

项目地址：

https://github.com/jonathandelgado/SublimeTodoReview

基本使用方法

  1. 安装 TodoReview，支持sublime text 3。
  2. Command + Shift + p 调出命令窗口，输入todo，选择 TodoReview:Open Files。
  3. n键移动至下一个，p上一个。回车直接跳转到对应的文件并定位到该行。
  4. 可以检测到的pattern有: TODO, NOTE, FIXME, CHANGED。

Rails 中的约定convention

发表于 2016-03-07 | 分类于 rails |

Convention Over Configuration
约定优于配置

单复数的约定

参考 Rails 约定汇总

Model用单数因为它表示一个对象如User，
数据库表用复数因为它存放的是对象的集合，
Controller用复数因为它是对对象集合的操作

Routes.rb中定义session一般用resource :session，而不是普通的resources :sessions。因为一般只会操作当前用户的session，不会操作所有session，所以不能定义为复数。

即如果一个请求一个资源时不需要指定ID，就在routes中用单数，如/profile显示当前登录用户的信息，这样你可以使用单数的/profile而不是/profile/:id。
也可以用match “profile” => “users#show”

User.all, users#show这样的表示。
其中的点.与井号#使用也是有约定的，点.用于调用类方法，井号#用于调用实例方法。

路由的约定

命名约定

变量名全部小写，名与名之间用下划线分开。

类和模板，每个短语（包括第一个词）开头都大写，不用下划线分割。

数据库表名,全部是小写字母，词与词之间用下划线，并且加s变复数

line_item.rb文件中，知道类的定义，类名LineItem

控制器命名约定，类名为StoreController,则这个类在store_controller.rb文件中，反过来也可懂。

类名为LineItemController，可以在app/view 下找到line_item文件夹的视图模板。

在控制器中的部分方法可在视图模版中找到对应的页面。

单复数约定

首字母大小写是无关紧要的, 最终产生的结果是一致的.

model一般都用单数, 复数的model会有歧义, 比如当你创建model的一个实例的时候, 会觉得你在创建多个实例

其它很多地方用复数形式, 比如数据表的名字, 比如routes中的url形式以及url的名称等等.

既然很多地方都用复数, 那么不如统一用复数形式. 另外经过检验, 统一用单数的方式有点行不通, 到restful routes那块跟Rails现有的设计有冲突. 那么索性就统一用复数形式好了.

文件名的单复数问题

经过观察, rails所有的常见的文件名都是小写的. 所以大小写的问题没有争议, 都用小写即可.
但是单复数问题不太统一, 比如users_contoller.rb, model/user_spec.rb.
这边总结的规律是:
跟model相关的东西, 单数居多. 比如model文件, model_spec文件等等
跟controller相关的东西, 复数居多. 比如 users_contoller文件, request/users_spec文件等等.
跟view相关的东西也是复数居多. 因为view的文件的创建是在创建contoller的时候自动创建的, 所以它跟contoller基本上是一致的.比如 javascripts/users.js.coffee
其它场合应该是单复数都可以.

最终的解决方案:

大小写问题已经没有争议了, 都用小写即可
单复数问题: model名字用单数, 其它场合都用复数

解决 "rails new projet" 需要输入密码的问题

发表于 2016-03-02 | 分类于 rails |

每次rails new 执行到bundle install 后总提示需要输入root密码，而且需要输入多次，很是麻烦。

以前解决方法是: sudo !!。

现在的方法：

$ rails new new_project --skip-bundle

让 rails new 后不直接运行 bundle install

进入 new_project 目录，运行：

$ bundle install --path vendor/bundle

这一句的意思是把 gems 安装到 vendor/bundle 下而不是系统的 gems 安装目录中。

如果你的 gem 镜像改为 ruby.taobao.org，可能会遇到有些包需要重新安装而这些包淘宝又没有，需要重新加 rubygems.org源。

$ gem source -add https://rubygems.org

似乎把问题变复杂了，而且文件夹大小也变大很多，但输入root密码并不是件很好的事，而且妈妈从小告诉我：东西归类是好件好事。
